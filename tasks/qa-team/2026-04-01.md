## Critical Bug Fixed This Cycle: 389efbf

### Root cause: fetchJSON had no User-Agent header
The NHL API blocks bare server requests (returns 403). `fetch()` in Next.js server
context sends no User-Agent by default. Every NHL pick resolution attempt from
Vercel has been silently returning null → "pending" since launch for picks that
needed boxscore data (team picks, period picks).

Client-side resolution worked because browsers always send a User-Agent.
This is why client visits resolved NHL ML picks but not NHL totals/period picks.

### Three fixes in 389efbf:
1. fetchJSON: added User-Agent header to all upstream fetches
2. resolveNHLTeamPick: "Team Points O/U" accepted alongside "Team Goals O/U"
   + pickLabel-based line extraction (so "STL Over 2.5 Goals" → line=2.5)
3. normalizeIncomingPick: "1P ML" betType inferred from /\b1p\s*ml\b/
   resolveNHLTeamPick: "1P ML" branch fetches play-by-play and counts P1 goals

### Known results post-fix:

- DAL 1P ML (3/29): P1 score 0-0 → PUSH
