### Root cause found: picks settlement has no server-side path

**Architecture finding (c839a5b):**
/api/picks/resolve is a POST endpoint called exclusively by the browser client
(usePicks.ts). The hook reads pending picks from localStorage, calls the API,
and if picks settle, writes back to Supabase via persistResolvedPickResults.

There is NO server-side settlement cron in vercel.json for pick_history.
The goose-model/auto-grade cron at 7 AM (rolling window from bf9ef8b) grades
goose_model_picks — a different table entirely. pick_history has been browser-only.

This explains every multi-day debt this cycle:
- 3/23 Jalen Suggs — 5 days (no browser traffic over weekend)
- 3/24 STL/CAR Goals — 4 days (same pattern)
- 3/27 CLE/Daniels/Siakam — 36h (low weekend traffic)
- 3/29 DAL 1P ML + 3 MLB picks — 44h+ and counting

**Fix: new /api/admin/picks/settle-pending endpoint (c839a5b)**
- Queries pick_history for result=pending within 3-day window
- Maps via mapPickHistoryRecordToAIPick (full snapshot, all leagues)
- Runs resolvePick() per league — NHL, NBA, MLB, PGA all covered
- Persists via updatePickResultsInSupabase
- Returns summary: checked/settled/still_pending/results
- Cron: 0 7,14,20 * * * (7 AM, 2 PM, 8 PM UTC — three server-side runs daily)
- vercel.json now has 14 crons

tsc --noEmit clean. Committed c839a5b, pushed to main. Vercel deploy in progress.
